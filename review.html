2.1 Review the ongoing use of IT tools and change approach as needed – in making

closing tags and I sometimes messed up because of this. Trello is something I never
really use but I can see the appeal of it from a planning standpoint.
2.2 Describe whether the IT tools selected were appropriate.
The programs I used where appropriate for the tasks we were given, Visual Studio Code

The strengths of the project I feel I could do most of the code through research and

could change it and if I had more time. The weaknesses of the project were that I

productivity as it would take off some stress of a big body of work, using slack can
also be good to keep upto date on what needs to be done and be in direct contact with
the team when not in the same room or at home.

The way I would test my website is to go over each page and check if they are correct
